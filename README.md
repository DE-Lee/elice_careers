## 2. 강의자료 및 실습 콘텐츠

4주 과정 중 **3주차 Day3 "RAG 파이프라인 구축 + 출처 표시"** 1일 분량을 제작했습니다.

### 하루 구성 (09:30~16:30, 6블록)

| 블록 | 시간 | 제목 | 슬라이드 |
|------|------|------|----------|
| 오전1 | 09:30~10:20 | 체험: 완성품 데모 | 18장 |
| 오전2 | 10:30~11:20 | 이해: 코드와 원리 | 16장 |
| 오전3 | 11:30~12:30 | 심화: 출처 표시 | 20장 |
| 오후1 | 13:30~14:20 | 실습: 기본 RAG 구축 | 11장 |
| 오후2 | 14:30~15:20 | 실습: 커스텀 + 출처 | 11장 |
| 오후3 | 15:30~16:30 | 회고: 퀴즈 + 토론 | 17장 |

**합계: 93장** / 설계 원칙: 체험 → 이해 → 심화 → 구축 → 개선 → 정리

### 실습 노트북

| 파일 | 용도 | 설명 |
|------|------|------|
| 오전1_시연.ipynb | 강사 시연 (블록1) | `ask("질문")` 한 줄로 RAG 결과 시연. 기본/커스텀 답변 비교(`compare`) 포함 |
| 오전3_시연.ipynb | 강사 시연 (블록3) | 출처 정확성 검증 데모. 답변과 PDF 원문을 나란히 대조 |
| 오후_실습.ipynb  | 수강생 실습 (블록4~5) | RAG 5단계 직접 구현. 커스텀 프롬프트·출처 표시 함수까지 단계별 안내 |

### 실습 환경

| 항목 | 사양 |
|------|------|
| Python | 3.9+ |
| 임베딩 | HuggingFace `sentence-transformers/all-MiniLM-L6-v2` (로컬, CPU) |
| LLM | Google Gemini (gemini-2.0-flash-lite / 2.0-flash-001 / 2.5-flash 자동 선택) |
| 벡터DB | ChromaDB |
| 주요 패키지 | langchain, langchain-huggingface, chromadb, pypdf, sentence-transformers |
| 실습 데이터 | 스마트냉장고 RF9000 제품사양서 + 시험성적서 (가상) |

### 실행 방법

1. [Google AI Studio](https://aistudio.google.com/apikey)에서 Gemini API 키 발급
2. 노트북의 `GOOGLE_API_KEY` 항목에 발급받은 키 입력
3. 셀 순서대로 실행
